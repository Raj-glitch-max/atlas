# Project Structure

Atlas uses a documentation-first repository foundation. This file is the canonical filesystem map for active repository organization.

## Directory Tree

```text
atlas/
|-- .claude/                         ignored local agent/session state
|-- .devcontainer/                   optional dev-container definition
|-- .git/                            git metadata
|-- .github/                         GitHub templates, Dependabot, workflows
|   |-- ISSUE_TEMPLATE/
|   `-- workflows/
|-- .vscode/                         editor recommendations/settings
|-- agents/
|   `-- agents/                      active reasoning framework root
|       |-- council/                 council reviewer files
|       |-- domain/                  domain-anchor files
|       |-- journal/                 decision memory
|       |-- templates/               agent/journal templates
|       `-- working/                 dynamic specialists and examples
|-- claude-skills/                   ignored third-party reference repo
|-- context/                         current project-state context
|-- lab/                             engineering research laboratory
|-- rfc/                             existing RFC-style archive candidates
|-- scripts/                         repository maintenance scripts
`-- tests/                           deferred test placeholder
```

Root markdown files hold project orientation, frozen planning documents, research outputs, and repository governance. `scripts/frozen-docs.list` is the source of truth for which root and lab documents are frozen.

## Directory Purposes And Ownership

| Directory | Purpose | Owner |
|---|---|---|
| `.claude/` | Local agent/session state. Ignored; not a repository artifact. | Local operator |
| `.devcontainer/` | Optional reproducible contributor shell using the root `Dockerfile`. | Repository maintainer |
| `.git/` | Git object database and local repo metadata. | Git |
| `.github/` | GitHub issue/PR templates, Dependabot, and repo-health CI. | Repository maintainer |
| `.vscode/` | Workspace editor recommendations only. | Repository maintainer |
| `agents/agents/` | Persistent reasoning framework, governance, council, domain anchors, journal, templates, working specialists. | Founder via `agents/agents/GOVERNANCE.md` |
| `claude-skills/` | Third-party acquired skills kept on disk for reference only. | Founder; ignored by git |
| `context/` | Current project mission, state, hypotheses, and backlog. | Founder via journaled decisions |
| `lab/` | Engineering research lab process, logs, evidence references, and experiment plans. | Lab roles in `lab/LAB_README.md` |
| `rfc/` | Existing RFC-style planning outputs. No new RFCs are created under the current repository-hygiene constraint. | Unknown until explicitly reclassified |
| `scripts/` | Repository maintenance scripts. | Repository maintainer |
| `tests/` | Placeholder for future stack-specific tests. | Repository maintainer until a stack is selected |

No `CODEOWNERS` file exists. Ownership here records the responsibility model already present in repository governance.

## Frozen Areas

Frozen files are listed in `scripts/frozen-docs.list` and hash-pinned by `FROZEN.sha256`. They include:

- the original frozen planning documents;
- the Phase 7 product-definition package;
- the Phase 8 engineering-requirements package;
- static lab governance files.

Do not edit frozen files unless `CONTRIBUTING.md` section 4 is followed.

## Generated Areas

| Path | Rule |
|---|---|
| `.git/` | Generated and managed by git. Never edit manually. |
| `.claude/` | Local generated session state. Ignored by git. |
| `FROZEN.sha256` | Generated by `make frozen-baseline`, but tracked because it is the freeze baseline. |
| `lab/evidence/` | Future experiment artifacts. Manifests and logs may be tracked; large binary captures are not tracked. |
| Standard build/cache outputs | Ignored by `.gitignore`: `node_modules/`, `dist/`, `build/`, `out/`, `target/`, `bin/`, `__pycache__/`, virtualenvs, object/class files, and env files. |

## Experimental Areas

| Path | Rule |
|---|---|
| `lab/` | Runs pre-registered engineering experiments only. It is not a product design or implementation area. |
| `tests/` | Placeholder until a language stack is selected. |
| `rfc/` | Existing RFC-style material only. Do not create new RFCs under current governance. |

## Ignored Areas

`.gitignore` intentionally ignores:

- `.claude/` local agent/session state;
- `claude-skills/` third-party reference repository with its own `.git`;
- OS/editor junk;
- stack-agnostic build and cache outputs;
- large evidence binaries and encrypted/signature artifacts.

Ignored files are local or externally managed. Important source files should not be added under ignored paths.

## Rules For Adding Directories

1. Use lowercase kebab-case directory names unless a well-known tool convention requires a different name, such as `.github/` or `.devcontainer/`.
2. Add a directory only when it has a clear purpose and owner.
3. Update this file in the same change that adds the directory.
4. Update `.gitignore` when the directory is generated, cache-like, local-only, or too large for git.
5. Do not add nested repositories unless they are intentionally ignored or intentionally configured as submodules.
6. Do not add duplicate wrapper names such as `name/name/` without an explicit migration plan.
7. Do not add new RFC directories or files unless the RFC policy in `DEVELOPMENT_RULES.md` changes first.
